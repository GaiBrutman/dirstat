A — Right-click context menu

Right-clicking any tile opens a small menu. No UI clutter; operations are hidden until needed. The most familiar pattern for desktop apps.

Pros

  • Zero visual clutter
  • Familiar to desktop users
  • Works on any tile size

Cons

  • Not discoverable on touch/trackpad
  • No hint actions exist

B — Hover overlay buttons

Small icon buttons appear in the corner of a tile on hover. Immediately actionable without an extra click, but only works on tiles large enough to fit the buttons.

Pros

  • One-click actions
  • Visually shows what's possible

Cons

  • Unusable on small tiles
  • Can feel cluttered on hover

C — Click to select + bottom action bar

Single-click selects a tile (dashed outline), revealing a persistent info + action bar at the bottom. Same pattern as Finder. Works on any tile size.

Pros

  • Works regardless of tile size
  • Shows name + size at a glance
  • Familiar Finder-like pattern

Cons

  • Requires two interactions (select, then act)
  • Takes vertical space
